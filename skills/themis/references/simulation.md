# Simulation Protocol

Themis simulation is not world prediction.

Simulation means testing whether the skill behavior works:
- dynamic role generation
- structured challenge
- conflict detection
- supervisor arbitration
- compact shared memory
- final report quality

## Required simulation cases

1. Software app
2. OSS security tool
3. Student/community product
4. Weird role request

## Pass criteria

A simulation passes if:

- roles fit the user task
- roles are functional, not fictional
- each role has mission and must_challenge
- every role report uses the required schema
- at least one useful challenge appears
- conflicts are resolved or escalated
- final supervisor report exists
- compressed state exists
- no empty agreement appears

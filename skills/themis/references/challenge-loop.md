# Challenge Loop

Themis exists to prevent blind agreement.

## Required challenge types

At least one role should challenge:

- the user request
- another role’s assumption
- the proposed implementation
- the evidence quality
- the testability of the plan
- the safety/risk boundary

If there is no meaningful challenge, the supervisor must explicitly state:

NO_BLOCKING_CONFLICT:
Reason:
Evidence:
Residual risk:

## No empty agreement rule

Forbidden:

“I agree.”
“Looks good.”
“No concerns.”
“Proceed.”

Allowed:

“I agree with the backend contract, but QA must test invalid payloads and timeout behavior.”

## Conflict card

Use this format:

CONFLICT_ID:
BETWEEN:
CLAIM_A:
CLAIM_B:
IMPACT:
EVIDENCE:
RESOLUTION:
